# 商品识别 AI 服务 · 使用说明

这是"商品识别"功能的 AI 推理服务（YOLO + FastAPI），**独立于 Docker 微服务运行**。有 NVIDIA 显卡更快，没有显卡也能用 CPU 跑（慢一些）。

## 一、环境准备

需要 **Python 3.12**（建议用 conda 或 uv 建独立虚拟环境）。

安装依赖：

```bash
# 1) PyTorch —— 有 N 卡装 GPU 版（CUDA 12.8）：
pip install torch torchvision --index-url https://download.pytorch.org/whl/cu128
#    没有 N 卡就装 CPU 版（去掉 --index-url 即可）：
#    pip install torch torchvision

# 2) 其余依赖：
pip install ultralytics fastapi "uvicorn[standard]" python-multipart pillow
```

## 二、模型权重

仓库的 `models/` 目录里已包含可直接用的权重：

| 文件 | 说明 |
|---|---|
| `models/snacks_11s_balanced.pt` | **部署默认模型**（YOLO11s + 过采样，约 18MB，mAP50≈0.994） |
| `models/snacks_11s_base.pt` | 对照基线（YOLO11s 原始） |

> YOLO12x 模型体积太大（450MB+），仓库未包含；服务启动时若发现某个权重缺失会**自动跳过**，不影响其余模型使用。想要 12x 可自行用 `train_balanced.py` 训练得到，放到 `models/snacks_12x_balanced.pt` 即可。

## 三、启动服务

```bash
python reco_service.py
```
启动后监听 **8123** 端口，会把存在的模型加载到 GPU（或 CPU）。验证：
```bash
curl http://localhost:8123/health
# {"status":"ok","device":"cuda:0","models":["11s_balanced","11s_base"]}
```

## 四、和前端对接

`Week1/shop-frontend/recognize.html` 里的 `RECO` 默认是 `http://localhost:8123`：
- **同一台电脑**跑：不用改，直接用。
- **算法层单独一台电脑**：把 `RECO` 改成那台电脑的局域网 IP（如 `http://192.168.1.50:8123`），并确保该机防火墙放行 8123、服务以 `0.0.0.0` 监听（已默认）。

## 五、接口

| 方法 | 路径 | 说明 |
|---|---|---|
| GET | `/health` | 健康检查，返回设备与已加载模型 |
| GET | `/models` | 已加载的模型列表（前端选择器用） |
| POST | `/recognize` | multipart 表单：`file`(图片) + `model`(模型key)，返回检测框+置信度 |

示例：
```bash
curl -F "file=@某张图.jpg" -F "model=11s_balanced" http://localhost:8123/recognize
```
