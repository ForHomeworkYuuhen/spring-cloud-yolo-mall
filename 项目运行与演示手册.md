# 项目运行与演示手册

> 微服务智能商城 · Spring Cloud Alibaba 后端 + 前端 + YOLO 商品识别

整个系统分两部分，**演示前两部分都要启动**：
- **后端微服务 + 前端**（Docker 一键起，免装环境）
- **商品识别 AI 服务**（算法层，独立 Python 进程，建议有 NVIDIA 显卡）

## 二、启动方式

### 1. 后端微服务全栈（Docker）

确保已启动 Docker Desktop，然后：

```powershell
cd E:\Code\Yueqian\Week1
.\start.ps1
```
（脚本会 `docker compose up -d --build`，等网关就绪，并自动处理"Nacos 注册抢跑"问题。也可手动 `docker compose up -d --build`。）

首次构建几分钟。完成后 9 个容器全部运行：mysql / nacos / sentinel / zipkin + 商品/订单/用户/网关 + 前端。

### 2. 商品识别 AI 服务（算法层）

```powershell
& "E:\Code\Yueqian\yolo-snacks\.venv\Scripts\python.exe" "E:\Code\Yueqian\yolo-snacks\reco_service.py"
```
监听 **8123**，自动加载 `yolo-snacks/models/` 下的模型权重（仓库已带 11s 部署模型）。

### 3. 商品推荐服务（个性化推荐，算法层）


首页"热门推荐"会自动调用它做个性化排序；**不启动也不影响首页**（自动按销量兜底）。

### 3b. AI 客服问答服务（知识图谱 + LLM，算法层）


### 4. 确认都起来了

## 三、演示清单（按知识点）

### 演示 1 · 首页与商品浏览
打开 http://localhost:8088，展示：搜索框、两个入口（商品识别 / 热门推荐）、商品列表。商品数据由网关从商品微服务取回。

### 演示 2 · 登录注册 + JWT 安全鉴权（安全亮点）
- 在首页点"登录/注册"，注册一个账号 → 登录成功后服务端签发 **JWT**（密码用 BCrypt 加盐存储）。
- 命令行验证 JWT 真实有效（注意 PowerShell 传 JSON 用文件避免引号问题）：
```powershell
[IO.File]::WriteAllText("$env:TEMP\r.json",'{"username":"demo","phone":"13800000000","password":"Pass1234"}')
$jwt = (curl.exe -s -X POST -H "Content-Type: application/json" --data "@$env:TEMP\r.json" http://localhost:9000/user/register | ConvertFrom-Json).token
curl.exe -s -o NUL -w "无token →%{http_code}`n"  http://localhost:9000/order/hi
curl.exe -s -o NUL -w "带JWT →%{http_code}`n"   -H "Authorization: Bearer $jwt" http://localhost:9000/order/hi
curl.exe -s -o NUL -w "篡改JWT →%{http_code}`n" -H "Authorization: Bearer ${jwt}XX" http://localhost:9000/order/hi
```
预期：无 token = 401、带有效 JWT = 200、篡改 = 401（验签拦截）。

### 演示 3 · 商品识别（图片上传走微服务 → 算法层）
- 首页点"商品识别" → 上传 `E:\Code\Yueqian\test_samples\` 里的图 → 页面框出零食 + 置信度。
- 链路：前端 → 网关 `/reco` → 商品微服务（应用层）→ 算法层 AI 服务 → 返回。命令行验证：
```powershell
curl.exe -F "file=@E:\Code\Yueqian\test_samples\sample1.jpg" -F "model=11s_balanced" http://localhost:9000/reco/recognize
```
返回里有 `image_w/image_h` 和检测框，证明图片真的传到算法层并识别了。

### 演示 4 · Nacos 配置中心 + 动态刷新（@RefreshScope）
算法服务地址 `reco.service.url` 存在 Nacos 配置中心，应用层动态读取：
1. 把 `Week1\nacos-config\shop-product-service.yaml` 上传到 Nacos 控制台（DataID `shop-product-service.yaml`，分组 DEFAULT_GROUP）；
2. 控制台修改 `reco.service.url` → 发布；
3. 访问 http://localhost:9000/reco/recognize/config → **不重启**就能看到地址变了（这就是 @RefreshScope 刷新）。

### 演示 5 · Sentinel 网关限流（课后扩展：自定义限流返回）
1. 登录 Sentinel 控制台（http://localhost:8080），左侧选 `shop-gateway` → 网关流控规则 → 新增；API 类型 Route ID 填 `shop-product-service`，QPS 阈值 1，保存；
2. 浏览器快速狂刷 http://localhost:9000/admin/products/1 ；
3. 超阈值返回自定义 JSON：`{"code":0,"message":"接口被限流了"}`（而非 Sentinel 默认提示）。

### 演示 6 · 网关局部过滤器·统计请求耗时（课后扩展）
经网关访问订单接口 http://localhost:9000/order/hi （需带 JWT），网关控制台会打印 `请求耗时: xx ms`；访问商品接口则不打印（局部过滤器只挂在订单路由）。看日志：`docker compose logs gateway`。

### 演示 7 · 链路追踪 Zipkin
访问几次经网关的接口后，打开 http://localhost:9411 → 查询 → 能看到一次请求在各微服务间的调用链与耗时（Sleuth 埋点 + Zipkin 展示）。

### 演示 8 · 个性化推荐（协同过滤，自动展示无按钮）
首页"热门推荐"区由推荐服务（:8124）按当前登录用户做**千人千面**排序，算法是基于天池用户行为数据训练的 **ItemCF + 类目级协同过滤**（见 `recommend/README.md`）。命令行验证：
```powershell
# 不同用户 -> 不同推荐顺序（个性化）
curl.exe "http://localhost:8124/recommend?user_id=alice&n=4"   # 例 {"item_ids":[4,5,6,1]}
curl.exe "http://localhost:8124/recommend?user_id=bob&n=4"     # 例 {"item_ids":[7,1,4,5]}
# 协同过滤本身：和某天池物品最相似的物品（喜欢A的人也喜欢B）
curl.exe "http://localhost:8124/similar?item_id=97655171&n=5"
```
页面演示：登录不同账号刷新首页，"热门推荐"四个商品的顺序随用户变化。

### 演示 9 · AI 客服（知识图谱 + LLM 问答）
任意页面右下角"AI 客服"悬浮窗，问商品相关问题，回答来自**商品知识图谱（neo4j）检索 + LLM 生成**（GraphRAG，不瞎编）。
1. 图谱：neo4j 浏览器 http://localhost:7474 里 `MATCH (n) RETURN n` 可看到 商品/品牌/类目/店铺 的图谱关系；
2. 命令行验证：
```powershell
curl.exe "http://localhost:8125/ask?q=Cheetoz有什么产品"
curl.exe "http://localhost:8125/ask?q=最便宜的零食是哪个"
curl.exe "http://localhost:8125/ask?q=你们卖辣条吗"   # 没有的商品会如实说明，不编造
```
回答里的 `facts` 是图谱检索到的真实商品，`source` 标明走的是 LLM 还是图谱模板。

## 四、常见问题

- **端口被占**（3307 / 8080 / 9000 等报 `port is already allocated`）：本机已有程序占了，改 `docker-compose.yml` 对应宿主机端口。
- **网关 500 / Nacos 里没服务**：业务服务抢跑了 Nacos 注册，`docker compose restart gateway product-server order-server user-server`（`start.ps1` 已自动处理）。
- **前端改了没生效**：浏览器缓存，硬刷新 Ctrl+Shift+R 或无痕窗口。
- **识别页显示"示例结果"**：识别 AI 服务（:8123）没启动，启动后即真识别。
- **热门推荐每个人都一样**：推荐服务（:8124）没启动，首页按销量兜底；启动后即按用户个性化。少数账号可能命中冷启动用户而退化成热度序，属正常。
- **没有 NVIDIA 显卡**：识别服务会自动用 CPU，慢一些但能跑。
